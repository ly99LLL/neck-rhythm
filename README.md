## 🎯 项目简介

**颈部律动**是一款零依赖、即开即玩的网页节奏游戏。它利用 **MediaPipe 人脸网格追踪** 实时检测玩家头部运动，将方向映射到游戏中——你不需要鼠标、不需要手柄，只需要转动你的脖子。

游戏提供两种截然不同的玩法机制、三种难度、摄像头/键盘双模式。单文件 HTML，无需安装编译，双击即玩。

| 属性 | 详情 |
|------|------|
| 类型 | 网页节奏游戏 |
| 文件 | 单文件 `index.html` ~1300 行 |
| 依赖 | 零本地依赖，MediaPipe CDN 按需加载 |
| 运行 | 双击打开 / 静态服务器 |
| 推荐浏览器 | Chrome / Edge |

## 💡 创作缘由

作为一名每天对着屏幕写代码 8-10 小时的开发者，我深切感受到**颈椎疲劳**带来的困扰。肩颈酸痛、头晕手麻，这些"职业病"几乎每个程序员都逃不掉。

市面上现有的解决方案要么枯燥（定时提醒做操），要么门槛高（颈椎康复训练视频），要么需要额外设备（VR 头显、体感手柄）。我很自然地想到：**能不能把颈部锻炼做成一个好玩的游戏？**

于是有了"颈部律动"——打开浏览器就能玩的头部节奏游戏。不占用额外时间（一局 90 秒），不需要任何安装，不需要额外设备（有摄像头更好，没有也能键盘玩），让颈椎在不知不觉中完成各方向的拉伸。

> **初心：让每一个久坐的人，都能在音乐和节奏中，给脖子一个放松的机会。**

## ✨ 创作灵感

项目的视觉和玩法设计受到多方启发：

### 🎨 视觉风格
- **合成波/赛博朋克美学** — 暗色背景 + 霓虹色调（青蓝 Cyan、品红 Magenta、金色 Gold），向 80 年代街机游戏和《Hotline Miami》《Katana ZERO》等独立游戏致敬
- **Orbitron 字体** — 科技感十足的游戏界面字体
- **径向汇聚设计** — 箭头从四面八方飞向中心的视觉呈现，类似音游的"下落式"但改为 360 度环向

### 🎵 玩法设计
- **《DJMAX》《osu!》** — 节奏游戏的核心爽感来源：精确命中、连击反馈、评分等级
- **《Beat Saber》** — 体感+方向判定的结合，将身体运动融入玩法
- **"都来都来"模式** — 传统音游逻辑：音符来了就接住
- **"别来沾边"模式** — 反向思维：箭头来了就要提前站在对立面挡住，类似"逆向思维"小游戏的创意

### 🏃 健康理念
- **《颈椎保健操》** — 上下左右 + 对角线的全方位颈椎运动
- **番茄工作法** — 90 秒一局，适合在 25 分钟工作间隙来一把，不耽误正事

## 🔥 核心特性

### 🎯 两种玩法机制

| | 🤸 都来都来 (Stretch) | 😈 别来沾边 (Rebel) |
|---|---|---|
| 规则 | 箭头进圈 → 同方向 → 吃掉 | 箭头到圈边 → 反方向 → 挡住 |
| 判定窗口 | 箭头在圈内任意时刻 | 箭头接触圈边那一瞬间 |
| 容错率 | 较宽松 | 零容错，没挡住直接 Miss |
| 适合 | 休闲放松 | 挑战自我 |

### 🎥 头部追踪
- 基于 **MediaPipe Face Mesh** 的实时人脸网格追踪
- 通过鼻尖偏移量（归一化到眼距）判断头部方向
- 自适应中立位置校准（3 秒快速校准）
- 低分辨率摄像头（320×240）确保低延迟
- 人脸丢失 350ms 内保持方向，避免误判

### ⌨️ 键盘模式
- WASD / 方向键双套控制
- 支持两键组合实现 8 方向（含对角线）
- 摄像头不可用时自动切换

### 🎵 程序化音乐
- **Web Audio API** 实时合成，无需音频文件
- Kick / Snare / Hi-hat / Bass 四轨节奏
- Bass 走和弦进行，每局略有变化
- 节奏随难度变化（轻松 95 BPM → 挑战 135 BPM）

### 📊 评分系统
- 三级判定：Perfect / Great / Good（按箭头距中心距离）
- 连击倍率：每 10 连击 +1 倍，最高 ×4
- 五级评分：S / A / B / C / D / F
- 粒子特效 + 判定文字 + 连击弹窗

### 🌐 零依赖部署
- 单文件 HTML，无构建工具
- 双击即玩（摄像头模式需要 Chrome/Edge 的 file:// 协议支持）
- 可直接部署到任何静态托管（GitHub Pages、Vercel、Netlify 等）

## 🕹️ 玩法说明

### 基本流程
```
选择模式 → 选择机制 → 选择难度 → 开始游戏
    ↓
摄像头校准（3秒） / 键盘直接开始
    ↓
倒计时 3 → 2 → 1 → GO!
    ↓
🎮 游戏进行中（80~100秒）
    ↓
📊 结算画面 → 再来一局 / 返回菜单
```

### 方向对照

| 方向 | 角度 | 颜色 | 标签 |
|------|------|------|------|
| ↑ 上 | -90° | 青蓝 #00e5ff | ↑ |
| → 右 | 0° | 品红 #ff2d95 | → |
| ↓ 下 | 90° | 金色 #ffd740 | ↓ |
| ← 左 | 180° | 紫 #b388ff | ← |
| ↗↘↙↖ | ±45°, ±135° | 各色 | (挑战模式) |

### 难度设定

| 难度 | BPM | 箭头密度 | 飞行拍数 | 圈半径 | 对角线 | 时长 |
|------|-----|----------|----------|--------|--------|------|
| 🌱 轻松 | 95 | 低 | 5 拍 | 130px | 无 | 100s |
| 🎵 标准 | 115 | 中 | 4 拍 | 110px | 8% | 90s |
| 🔥 挑战 | 135 | 高 | 3 拍 | 90px | 20% | 80s |

## 🛠️ 技术栈

| 技术 | 用途 |
|------|------|
| **HTML5 Canvas** | 游戏渲染引擎（径向网格、箭头绘制、粒子系统） |
| **MediaPipe Face Mesh** | 实时人脸 468 点网格追踪，头部方向识别 |
| **Web Audio API** | 程序化音乐合成（Kick/Snare/Hi-hat/Bass 四轨） |
| **Vanilla JavaScript (ES6+)** | 全部游戏逻辑，Class-based 架构 |
| **CSS3** | UI 界面、动画、响应式布局 |
| **Google Fonts (Orbitron)** | 科技感游戏字体 |
| **jsDelivr CDN** | MediaPipe 模型文件分发 |

> **零框架、零构建、零后端。** 纯前端单文件应用。

## 🏗️ 项目架构

```
App (顶层调度器)
├── AudioEngine        — Web Audio API 程序化音乐
│   ├── Kick (正弦波 150→40Hz)
│   ├── Snare (噪声 + 带通滤波)
│   ├── Hi-hat (噪声 + 高通滤波)
│   └── Bass (锯齿波 + 低通滤波，和弦进行)
│
├── Game               — 核心游戏引擎 (Canvas)
│   ├── 箭头生成 (从边缘向中心飞行)
│   ├── 命中判定 (都来都来 / 别来沾边 两种逻辑)
│   ├── 评分 & 连击系统
│   ├── 粒子特效 (ParticleSystem)
│   └── UI 渲染 (径向网格、状态环、方向指示器)
│
├── HeadTracker        — MediaPipe 头部追踪
│   ├── 人脸检测 & 网格提取
│   ├── 鼻尖-眼心偏移量计算 (归一化)
│   ├── 方向映射 (8方向 atan2)
│   ├── 自适应中立校准
│   └── 人脸丢失容错 (350ms)
│
├── KeyboardInput      — WASD/方向键输入
│   └── 单键/双键组合 → 方向 ID
│
├── BeatMapGenerator   — 程序化谱面生成
│   ├── 基于 BPM 的节拍网格
│   ├── 密度/变体/对角线概率控制
│   └── 最小间隔约束 (0.4拍)
│
└── UIManager          — 屏幕切换 & HUD
    ├── 加载 → 菜单 → 校准 → 游戏 → 结算
    └── PIP 摄像头小窗 / 键盘提示
```

### 数据流（每帧）
```
KeyboardInput.getDirection() ─┐
HeadTracker.currentDirection ─┤ → App._gameLoop()
                                ├── Game.update(dt, headDir)
                                │   ├── 生成箭头 (spawn)
                                │   ├── 更新位置 (fly toward center)
                                │   ├── 命中判定 (stretch / rebel)
                                │   └── 粒子更新
                                └── Game.draw()
                                    ├── 径向网格背景
                                    ├── 箭头绘制 (带拖尾)
                                    ├── 粒子渲染
                                    └── HUD 更新
```

## 🚀 快速部署

```bash
# 用 Chrome 或 Edge 直接打开 index.html
# 摄像头模式需要 file:// 协议（Chrome/Edge 支持）
```

### 方式二：本地服务器
```bash
# Python
python -m http.server 8080

# Node.js (npx)
npx serve .

# 然后浏览器打开 http://localhost:8080
```

```bash
git init
git add index.html README.md
git commit -m "Initial commit: 颈部律动 Neck Rhythm"

git push -u origin main
```

## 💪 项目意义

### 健康价值
长时间低头工作是颈椎病的首要诱因。颈部律动将枯燥的颈部锻炼包装成有趣的节奏游戏，**降低运动门槛、提升参与意愿**。每次 90 秒的游戏中，玩家会不自觉地完成数十次各个方向的颈部活动——这本质上是一次完整的颈椎拉伸。

### 教育价值
项目展示了如何在纯前端实现一个完整的游戏闭环：
- **计算机视觉**：MediaPipe 人脸追踪的实战应用
- **音频编程**：Web Audio API 的合成调度
- **游戏开发**：Canvas 渲染、粒子系统、谱面生成
- **UI/UX 设计**：游戏化体验、用户引导、错误处理

### 社会价值
开源、零成本、零门槛。任何有电脑的人都能用——无论是办公室白领、学生、还是居家办公者。无需付费、无需注册、无需安装。

## 📈 个人能力提升

通过独立完成这个项目，我在以下方面获得了显著成长：

### 🎮 游戏设计能力
- 学会了从玩家体验角度设计**判定容错机制**（都来都来有容忍窗口，别来沾边零容错——差异化的难度曲线）
- 掌握了节奏游戏中**谱面生成算法**（基于节拍网格的概率生成 + 最小间隔约束）
- 理解了 **"流畅状态"（Flow State）** 如何在游戏中营造——难度递增、即时反馈、清晰目标

### 👁️ 计算机视觉实践
- 深入理解了 MediaPipe Face Mesh 的 468 点人脸网格及其坐标系
- 掌握了**头部姿态估计**的简化方法（鼻尖-眼心偏移量归一化），无需完整的 3D 旋转矩阵
- 学会了处理真实场景的容错问题：人脸丢失、抖动抑制、自适应校准

### 🔊 Web Audio 编程
- 从零学习了 Web Audio API 的**程序化音频合成**（Oscillator、BiquadFilter、AudioBuffer）
- 掌握了音频事件调度（`currentTime` 时间轴）和实时节拍同步
- 理解了打击乐合成（噪声包络 → 滤波）、贝斯合成（锯齿波 → 低通）等音频设计基础

### 🏗️ 前端架构能力
- 设计了一个清晰的 **Class-based 模块化架构**，在单文件中保持了良好的关注点分离
- 实践了**异步流程控制**：摄像头初始化超时、校准超时、用户中断、自动降级
- 掌握了 Canvas 渲染的性能优化（devicePixelRatio 适配、对象池复用）

### 🎨 UI/UX 设计
- 建立了统一的**视觉设计语言**（合成波风格、霓虹配色、Orbitron 字体）
- 实现了**无感错误处理**：用户拒绝摄像头 → 静默切换键盘，不中断游戏流程
- 设计了**完整的用户旅程**：加载 → 模式选择 → 校准 → 游戏 → 结算 → 复玩
